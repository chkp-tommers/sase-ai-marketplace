Cursor Guide

Your complete guide to getting started and mastering Cursor's AI-powered development environment.

Overview

Cursor is an AI-powered code editor designed to help you code faster and smarter by deeply understanding your codebase. With Cursor, you can describe what you want to build or change in natural language, and the editor will generate, refactor, or review code for you. Cursor's AI is context-aware, meaning it leverages your entire project, coding standards, and even your team's shared knowledge to provide highly relevant suggestions.

  • Natural language code generation and refactoring
  • Context-aware completions and suggestions
  • Support for advanced workflows with MCPs (Model Context Protocol), rules, memories, and agent automation
  • Works with all major programming languages and supports extensions

Cursor is designed for both individual developers and teams, making it easy to onboard, share context, and collaborate efficiently.

Installation & Setup

  • Download Cursor: Visit cursor.com/downloads and select the installer for your operating system (Windows, Mac, or Linux).
  • Run the Installer: Follow the installation prompts. Once installed, launch Cursor.
  • First-Time Setup:
    • Shortcuts: Choose keyboard shortcuts that match your previous editor (e.g., VS Code, JetBrains).
    • Theme: Pick a light or dark theme, or customize your own.
    • Terminal Preferences: Set up your preferred shell and terminal settings.
    • Account: Sign up for a Cursor account to unlock all AI features. You can do this during setup or later in Settings.
    • Import Settings: If you're migrating from another editor, you can import your settings and preferences for a seamless transition.
Tips
  • You can revisit the setup wizard anytime by pressing Ctrl+Shift+P and searching for Cursor: Start Onboarding.
  • Cursor supports all major programming languages out of the box, and you can add more via extensions.

See the official installation guide for more details.

Core Concepts

  • Codebase Indexing: When you open a project, Cursor automatically indexes your codebase. This process allows the AI to understand your project's structure, dependencies, and logic, resulting in highly relevant suggestions. Indexing is fast (1â€“15 minutes depending on project size) and can be shared across teams for consistency.
  • Context Awareness: Cursor uses not just your open files, but the entire project context, including rules, memories, and recent changes, to inform its suggestions. This means the AI can reference related files, follow your coding standards, and even remember project goals.
  • AI Models (LLMs): Cursor supports multiple large language models (LLMs), each with different strengths. You can select the best model for your task (see next section).
  • Team Collaboration: Teams can share codebase indexes, rules, and memories, making onboarding and collaboration seamless.
For more details, see Cursor Concepts and Codebase Indexing.

LLMs in Cursor

Cursor lets you choose from a range of powerful LLMs, including GPT-4o, GPT-4, GPT-3.5, Claude, Gemini, and more. Each model has unique strengths for different coding and chat tasks.

  • GPT-4o: Fast, balanced, and strong at both code and chat. Great for general coding, documentation, and pair programming.
  • Claude: Excellent for long-context tasks and summarization.
  • Gemini: Lightweight and fast, ideal for quick iterations.
Example: Switch LLMs 
// Open the command palette
Ctrl+Shift+P
// Search for:
Cursor: Select Model
// Choose your preferred LLM (e.g., GPT-4o, Claude, Gemini)
For more details, see Cursor Models.

Prompt Engineering

Use rules, memories, and @-symbols to guide Cursor's AI and automate context-aware workflows. Combine these for advanced prompt engineering and team-wide standards.

  • Rules: Define project or team-wide coding standards, agent behaviors, and best practices. Rules can be set globally or per project and are automatically referenced by the AI when generating or reviewing code.
  • Memories: Persist important context, decisions, or facts for agents and AI. Memories help the AI remember project goals, architectural decisions, or recurring issues, making its suggestions more relevant over time.
  • @-Symbols: Use @prompt, @rule, or @memory in chat or commands to quickly reference and invoke context, templates, or tools.
Example: Apply Rules, Prompts, and Memories 
@prompt code-review
@rule typescript-strict
@memory project-goals
Example: Rule File Structure 
---
description: TypeScript strict mode for all code
alwaysApply: true
---
# TypeScript Strict

## Compiler Options
```json
{
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true
  }
}
```

## Coding Standards
- Use explicit type annotations
- Handle null/undefined cases
Example: Memory File Structure 
---
description: Project goals and context
---
# Project Goals
- Build a scalable, maintainable API
- Follow strict TypeScript standards
- Prioritize security and performance
For more details, see Rules, Memories, and @-Symbols.

MCP Integration

Model Context Protocol (MCP) allows Cursor to connect with external tools, data sources, and services, greatly expanding its capabilities for automation and advanced workflows.

  • Configure MCP servers in your project or workspace settings.
  • Restart Cursor after updating your MCP configuration.
  • Agents and tools can now access these MCP servers for advanced workflows (e.g., file operations, git actions).
Example: MCP Server Configuration 
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["@filesystem/mcp"],
      "env": { "ALLOWED_PATHS": "/workspace" }
    },
    "git": {
      "command": "git-mcp",
      "args": [],
      "env": {}
    }
  }
}

Agent Features

Agents in Cursor are AI-powered assistants that can automate tasks, refactor code, perform reviews, and more. Customize agent behavior with rules, prompt templates, and MCP tools.

  • Task Automation: Agents can break down complex tasks into steps, plan, and execute them in sequence.
  • Code Review: Agents can review code for style, security, and performance, using your project's rules and memories.
  • Refactoring: Agents can suggest and apply code improvements, including inline edits and terminal commands.
  • Modes: Agents support different operational modes (e.g., code generation, review, refactoring), each customizable with prompts and rules.
  • Tools: Agents can access external tools and data via MCP servers.
Example: Invoking Agent Features 
// Open the command palette
Ctrl+Shift+P
// Run:
Cursor: Plan
Cursor: Review
Cursor: Refactor
// Or use @prompt, @rule, @memory in chat to customize agent behavior
For more details, see Agent Overview, Agent Modes, and Agent Tools.

  • Review official docs for troubleshooting and advanced workflows.
