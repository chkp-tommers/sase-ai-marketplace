Claude Guide

Comprehensive guide for using Anthropic Claude: setup, LLMs, prompt engineering, MCP, and best practices.

Overview

Claude is Anthropic's advanced AI assistant, designed for safe, helpful, and honest collaboration on a wide range of tasks. Claude can write, review, and refactor code, answer questions, summarize documents, and automate workflows. It is accessible via API, SDKs, and the Anthropic Console, and supports advanced features like tool use, prompt engineering, and Model Context Protocol (MCP) integration.

  • Anthropic Home
  • Overview

Installation & Setup

  • Prerequisites: Anthropic Console account, API key, Python 3.7+/TypeScript 4.5+/Java 8+.
  • SDKs: Install via pip install anthropic (Python) or npm install @anthropic-ai/sdk (TypeScript). Java and Go SDKs also available.
  • API Key: Set ANTHROPIC_API_KEY as an environment variable.
  • First Steps: Use the Anthropic Console Workbench to experiment with prompts and responses before integrating into your application.

Core Concepts

Context Windows

Claude supports very large context windows (up to 200K tokens in Claude 3 Opus), allowing you to provide long documents, codebases, or conversations as input. This enables the model to reason over extensive information and maintain context across complex tasks. Use this feature to include all relevant background, requirements, or code in your prompts for more accurate and holistic responses.

// Example: Provide a long document as input
"Please summarize the following document: [paste document here]"

Reference: Context Windows

Prompt Caching

Prompt caching allows you to reuse previously processed prompts and their outputs, reducing latency and cost for repeated or similar requests. This is especially useful for applications with recurring queries or workflows. By caching results, you can deliver faster responses and optimize resource usage.

// Example: Cache a prompt
"Summarize the following article in three bullet points: [paste article here]"

Reference: Prompt Caching

Extended Thinking

Extended thinking enables Claude to reason through complex, multi-step problems by encouraging the model to "think out loud" and break down its process. This is particularly effective for tasks that require analysis, planning, or logical deduction. Use prompts like "Explain your reasoning step by step" to activate this capability.

// Example: Ask Claude to think step by step
"Let's think step by step: What are the pros and cons of each approach?"

Reference: Extended Thinking

Batch Processing

Batch processing allows you to send multiple prompts in a single API call, improving throughput and efficiency for large-scale or high-volume applications. This is ideal for scenarios like document summarization, bulk classification, or mass content generation.

// Example: Send multiple prompts
"Summarize the attached PDF and extract all key dates."

Reference: Batch Processing

PDF & File Support

Claude can process and extract information from PDFs and other file types. This enables workflows like document analysis, data extraction, and summarization directly from uploaded files. You can upload files via the API and instruct Claude to read, summarize, or answer questions about their content.

// Example: Upload a PDF and ask Claude to summarize
"Summarize the attached PDF and extract all key dates."

References: PDF Support, Files

Search Results & Sheets Integration

Claude can fetch and summarize web search results and work with Google Sheets. This is useful for research, data analysis, and automation. You can instruct Claude to pull in external data, process it, and return structured outputs or insights.

// Example: Search for the latest news on AI regulation
"Search for the latest news on AI regulation"

References: Search Results, Claude for Sheets

Opus is the most capable, with the largest context window and best reasoning, while Haiku is optimized for speed and cost. Choose the model that best fits your use case, balancing performance, cost, and context needs.

Best Practices in Prompt Engineering

Be Clear and Direct

Effective prompts are clear, direct, and unambiguous. Specify exactly what you want the model to do, avoid vague language, and use explicit instructions. For example, instead of "Summarize this text," say "Summarize the following article in three bullet points." This approach reduces misunderstandings and improves output quality.

// Example Prompt
"Summarize the following article in three bullet points: [paste article here]"

Reference: Be Clear and Direct

Prompt Templates & Variables

Prompt templates allow you to standardize instructions and inject variables for dynamic content. This ensures consistency and adaptability across different tasks and users. For example, a template for code review might include placeholders for the language, style guide, and code snippet. Use variables to make prompts reusable and maintainable.

// Example: Use a prompt template
"Summarize the following article in three bullet points: [paste article here]"

Reference: Prompt Templates & Variables

Multishot Prompting

Multishot prompting involves providing several input/output examples within your prompt. This helps the model learn the desired pattern and improves accuracy, especially for complex or nuanced tasks. For example, show a few Q&A pairs before asking a new question. This technique is powerful for classification, extraction, and structured outputs.

// Example: Use multishot prompting
"Summarize the following article in three bullet points: [paste article here]"

Reference: Multishot Prompting

Chain-of-Thought (CoT) Reasoning

Encourage the model to reason step by step by explicitly asking it to "think out loud" or break down its reasoning. This is especially useful for complex problem-solving, math, or logic tasks. For example, "Let's think step by step..." prompts the model to show its work, leading to more accurate and transparent answers.

// Example: Use chain-of-thought reasoning
"Let's think step by step: What are the pros and cons of each approach?"

Reference: Chain-of-Thought

Use XML Tags for Structure

Use XML tags or similar markup to enforce structure and clarity in model responses. For example, wrap answers in tags or use for each reasoning step. This makes parsing and post-processing easier and ensures the output format is consistent.

// Example: Use XML tags for structure
Summarize the following article in three bullet points: [paste article here]

Reference: Use XML Tags

System Prompts & Role Assignment

Assign a role or persona to the model using a system prompt. For example, "You are a world-class technical writer. Respond in clear, concise English." This helps guide the model's tone, style, and expertise, making outputs more relevant to your use case.

// Example: Assign a role to Claude
"You are a world-class technical writer. Respond in clear, concise English."

Reference: System Prompts

Prefill Claude's Response

You can seed the model's output with a starting point or partial answer. This is useful for drafting emails, code, or documents where you want the model to continue or complete a specific section. Prefilling helps maintain context and style.

// Example: Prefill Claude's response
"Dear [Customer's Name],

Thank you for reaching out. I'm here to help with your order. I've checked the status of your order and it's on its way to you. I'll keep you updated on the delivery.

Best regards,
[Your Name]"

Reference: Prefill Claude's Response

Chaining Prompts

Break complex workflows into multiple, linked prompts. For example, first ask the model to summarize a document, then use that summary as input for a follow-up task. Chaining prompts allows for modular, stepwise automation and better control over multi-stage processes.

// Example: Chain prompts
"Summarize the attached PDF and extract all key dates."

Reference: Chain Prompts

MCP Integration & Tool Use

MCP Connector

The Model Context Protocol (MCP) connector allows Claude to interact with external tools, data sources, and services. By configuring MCP connectors, you can enable advanced workflows such as file access, code execution, and more. This integration is essential for building powerful, context-aware AI applications.

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["@filesystem/mcp"],
      "env": { "ALLOWED_PATHS": "/workspace" }
    }
  }
}

Reference: MCP Connector

Remote MCP Servers

Remote MCP servers allow you to run tool integrations on separate infrastructure, improving scalability and security. Configure your LLM or agent to connect to these servers for distributed tool use. This is especially useful for organizations with strict security or resource requirements.
